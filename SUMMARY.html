System Architecture Summary

Objective: Transmit a live media stream (160-byte frames every 20ms) across a hostile network relay while maintaining a deadline miss rate of ≤ 1.0% and a bandwidth overhead of ≤ 2.0x.

1. Hybrid Error Recovery Design

To combat both isolated packet drops and sustained burst losses, the system uses a hybrid approach:

  • Forward Error Correction (FEC): The sender groups packets into pairs (e.g., Packet 0 and Packet 1) and calculates an XOR parity packet. If exactly one packet in the sequence drops, the receiver instantly mathematically rebuilds the missing payload using the parity packet. This provides zero-latency recovery for uniform packet loss.
  • Automatic Repeat reQuest (ARQ): If burst losses defeat the FEC parity, the receiver identifies the gap in its sequence buffer and generates a NACK via the feedback channel (port 47003).

2. Receiver & Jitter Buffer Implementation

The receiver operates asynchronously using a select() loop with a 10ms timeout.

  • Immediate Forwarding: Because the harness player calculates playout delay at the end of the run based on the strict formula T0 + DELAY_MS + (i * 20ms), valid packets are forwarded instantly upon receipt.
  • Aggressive NACK Throttling: The receiver actively sweeps for dropped frames and mathematically calculates which packets are still "alive" based on the deadline. It throttles NACK requests to once every 25ms per missing frame to prevent bandwidth exhaustion.

3. The "Double-Tap" Retransmission Strategy

Hostile network profiles introduce severe burst losses that frequently consume retransmissions. To counter this, the sender employs a "Double-Tap" mechanism.

  • Whenever a NACK is received, the sender queries its history buffer and transmits the requested frame twice in rapid succession.
  • This trades unused bandwidth budget for a massive increase in delivery probability. The system successfully reached an optimal 1.98x bandwidth overhead, leveraging maximum allowable redundancy to push the miss rate down to 0.93% at a 180ms delay.
