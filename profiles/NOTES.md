Our system achieves a VALID run on Profile B at a delay of 180ms. The architecture uses a hybrid protocol combining XOR Forward Error Correction (FEC) and an aggressive Automatic Repeat reQuest (ARQ) implementation. To combat isolated drops, the sender generates a parity packet for every consecutive pair of data packets, allowing the receiver to mathematically reconstruct a single dropped packet with zero round-trip latency. For heavy burst losses that destroy both data and parity packets, the receiver falls back to a 25ms-throttled NACK system. Because hostile networks often drop retransmissions, the sender utilizes a "double-tap" strategy, blindly firing two copies of any NACKed packet. This design explicitly trades unused bandwidth capacity for reliability, maintaining a strict footprint just below the 2.0x overhead limit. The system ultimately breaks when network burst drops are sustained longer than the 180ms deadline window, or when pure retransmission volume exceeds the 2.0x bandwidth cap.
